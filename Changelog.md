# BridgeTTS 更新履歴

---

## そのうちやるかもリスト

- **マルチセッション対応** — 複数端末から同時接続した際に、キャラクター選択・会話履歴・システムプロンプト設定・ログセッションをブラウザセッション単位で独立管理する。速度面で並列処理の向上は期待しないが、「別の端末で別のキャラと別々にログを残す」用途に対応できる。
  - `AppState` とログセッションをセッションIDキーの辞書で管理する形に変更
  - アクティブなシステムプロンプトファイル／スロット番号もセッション状態に含める
  - スロットの保存テキスト・`prompts/` ファイル群・LLM/TTS設定はグローバル共有のまま
- **ログをシナリオとして保存** — 既存の「シナリオを読み込む」（ユーザー発言を順次再送する機能）と連携する形で、会話ログをシナリオファイルとして書き出す機能。現在はログファイルをそのままシナリオとして読み込む仕様だが、ユーザー発言のみを抽出した専用フォーマットで保存することで管理しやすくする。

## 2026-04-25

### 機能追加
- LLM サーバのプリセットに「llama.cpp server」を追加（デフォルトポート: 8080）
- 環境設定の LLM サーバ設定にモデルアンロードボタンを実装。LM Studio / Ollama / text-generation-webui のプリセット選択時のみ表示される。アンロードの仕組みはサーバごとに異なり、LM Studio は管理 API (`POST /api/v1/models/unload`、`instance_id` 指定)、Ollama は `keep_alive: 0` による解放、text-generation-webui は `/v1/internal/model/unload` を使用する
- **テキスト読み上げ機能**を実装。ログメニューの「…」→「テキストを読み上げる」からテキストファイル（UTF-8 / Shift-JIS / EUC-JP）またはテキスト直接入力で、現在のキャラクターの声で順次 TTS 再生できる
  - プロデューサー／コンシューマー非同期方式により TTS 生成と再生を並列化。前の吹き出しを再生している間に次の音声を生成する
  - 読み上げ前にプレビューモーダルで内容確認・設定調整が可能。設定は自動保存される
  - 「1吹き出しの文章数」スライダー（1〜5）で1吹き出しに含める文の数を指定
  - 「送出文字数上限」で TTS API への1回あたりの最大文字数を設定（デフォルト1024、最小500、最大4096）。上限を3回連続で超えた場合はシステムメッセージを出して停止する
  - 「読み上げ時に括弧文字を除去する」チェックで指定した括弧文字（デフォルト: `()（）`）を除去してから TTS に送出する。括弧内のテキストは保持されるため読み上げ対象になる。通常チャットには影響しない
  - プレビューモーダルの「デフォルト設定」ボタンで文章数・文字数・括弧除去設定を一括リセットできる
  - 読み上げ開始時にメッセージ画面とログをクリアしてから開始する
  - 読み上げ中はログ自動保存を抑制しメモリ上に保持。読み上げ後もログ保存ダイアログから手動保存可能
  - 各吹き出しに ↻（再生成）・⋯（リプレイメニュー）・▶（再生）ボタンを付与。通常チャットと同じ操作が行える
  - 再生中の吹き出しの ▶ ボタンに外周スピニングリングを表示（通常チャットと統一）

- サイドバーの音声セクションに「生成ファイルを自動削除」チェックボックスを追加。ON（デフォルト）にすると TTS 生成ファイルを base64 エンコード後に自動削除する。OFF にするとファイルが Irodori-TTS の出力フォルダに残るため、出力ファイルを利用したい場合に使用できる。設定は `bridge_settings.json` に保存される

### 修正
- プリセット変更時にモデル一覧ドロップダウンが非表示にリセットされなかった問題を修正（内部変数名の誤りを修正）
- テキスト読み上げ吹き出しで ↻ 再生成を行うとメニューボタン・再生ボタンが増殖する問題を修正
- テキスト読み上げの ↻ 再生成中にスピナーが表示されない問題を修正
- 複数の設定を同時に保存した際に環境設定が初期化されることがある問題を修正。設定ファイルの書き込みにスレッドロックとアトミック書き込み（一時ファイル経由の `os.replace`）を導入し、同時書き込みによる JSON 破損を防止した

## 2026-04-23

### 機能追加
- LLM サーバ設定を刷新。環境設定から LM Studio / Ollama / text-generation-webui / KoboldCpp などのプリセットを選択でき、ホスト・ポート・パス・API キー・モデルをフィールドごとに指定・保存できるようになった
- LLM のモデル選択に対応。🔄 ボタンでサーバから利用可能なモデル一覧を取得し、入力補完または選択できる。Ollama など、モデル名の指定が必要なサーバで有用
- LLM / TTS の接続設定を永続化。従来は再起動で元に戻っていたが、設定ファイル（`bridge_settings.json`）に保存されるようになった
- TTS サーバ設定もホスト・ポートの個別指定に対応し、同様に永続化
- 環境変数（`LLM_API_URL` / `TTS_API_URL`）が設定されている場合、環境設定のチェックボックスで環境変数を優先使用するよう切り替えられる

### 修正
- iOS Safari でキーボード表示時に画面全体が上にずれて入力バーが消える問題を修正。`visualViewport.height` を使って `#app` の高さをキーボード開閉に追従させるよう変更
- iOS Safari でフォーム要素にフォーカスすると自動ズームが発生し、キーボード収納後も画面が横にはみ出す問題を修正。モバイル時にすべての入力欄の `font-size` を 16px 以上に強制することで自動ズームを抑止
- iOS Safari を含むモバイル環境でレイアウト全体（吹き出し・入力バー・ヘッダーのインジケーター等）が右にはみ出す問題を修正。グリッドの列定義を `1fr` から `minmax(0, 1fr)` に変更し、`min-width: auto` によるトラック拡張を防止。`#messages` と `#input-bar` に `min-width: 0` / `overflow: hidden` を追加

## 2026-04-22

### 機能追加
- 会話ログエリアの上端・下端をタップ/クリックすると最初のメッセージ・最後のメッセージへ一気にスクロールする機能を実装。スクロール余地がある方向にのみ帯が表示される
- キャラクターのリファレンス音声として `.mp3` ファイルを受け入れるよう対応。`.wav` と `.mp3` の両方がある場合は `.wav` を優先して使用する
- キャラクター編集画面に「MP3→WAV 変換」ボタンを実装。キャラクターフォルダに `.mp3` が存在し、環境設定で ffmpeg が有効な場合に押せるようになる。既に `.wav` がある場合は上書き確認を表示する（zip 形式のキャラクターは対象外）
- 会話ログの自動保存・手動保存・読み込み機能を実装。送信バー左の「…」ボタンからログの保存・読み込みができる
- 環境設定に「チャットログを自動保存する」チェックボックスを追加。OFF にするとログファイルへの書き込みを停止し、ON に戻したタイミングでそれまでの会話を含む全文をまとめて書き出す
- ログはキャラクター選択後にユーザーが最初のメッセージを送った時点でファイルを作成する（挨拶のみで終了した場合はファイルを残さない）
- 「ここからリプレイ」を改善。音声未生成の吹き出しがある場合、随時 TTS 再生成しながら順番に再生する。読み込んだログの吹き出しも対象になる
- ログ読み込み時に ↻（再生成）ボタンと簡易 ⋯ メニュー（ここからリプレイ）を吹き出しに追加する
- ログメニューに「シナリオを読み込む」を追加。選択したログのユーザー発言を現在のキャラクターへ順次送出して新しい会話を生成する。再生中は送信ボタンが ■ に変わり、クリックで停止できる

### 修正
- スクロール機能追加後にキャラクターを選択しても挨拶・メッセージ送信・背景描画が行われなくなる問題を修正
